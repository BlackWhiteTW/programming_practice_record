# 汽車維修估價單管理系統說明文件

## 一、問題領域（Problem Domain）描述

本系統對應的真實世界情境為「汽車維修廠估價單管理」

### 1. 主要物件

- **ServiceQuote（估價單）**：
  包含客戶姓名、客戶稱謂/職稱、估價日期、車輛品牌/型號/年份、服務內容、零件費、工資、稅率
- **QuoteManager（估價單管理器）**：
  管理多筆估價單 提供新增、刪除、查詢、列出等功能

### 2. 主要事件流程

- 建立估價單：輸入資料後系統自動計算稅金與總金額
- 查詢估價單：manager 可用「客戶姓名」或「估價單日期」作為關鍵字查詢
- 刪除估價單：manager 可隨時刪除指定編號估價單
- 瀏覽清單：顯示目前所有估價單

### 3. 業務結果與紀錄項目

- 每張估價單都會有系統編號（quoteId）
- 每張估價單都可顯示完整摘要（小計、稅金、總估價）

## 二、程式設計架構（Object-Based）

系統採用物件式設計（object-based）

### 1. 核心資料與邏輯

- 檔案：`QuoteSystemCore.h` / `QuoteSystemCore.cpp`
- 內容：

  #### 主要控制涵式
  - `ServiceQuote`：封裝估價單資料與金額計算
  - `QuoteManager`：封裝多筆估價單資料管理（新增、刪除、查詢）

  #### 輸入控制涵式
  - `trim`：字串處理工具 可以去除 \t、\r、\n
  - `toLowerAscii`：把輸入的大小寫固定成小寫
  - `containsText`：檢查是否包含 keyword 如果 keyword 為空 則試為符合
  - `promptLine`：輸入一行字串 可以包含空格 allowEmpty = false 則強制輸入非空內容
  - `promptInt`：整數輸入功能 可要求使用者輸入在範圍內的有效整數
  - `promptDouble`：浮點數輸入功能 可要求使用者輸入大於最小值的數值

### 2. 使用者介面層

- 檔案：`QuoteSystemUI.h` / `QuoteSystemUI.cpp`
- 內容：
  - `printMenu`：顯示主選單
  - `createQuoteFromInput` :創建估價單物件
  - `printQuotes`：顯示特定估價單查詢的結果
  - `printAllQuotes`：顯示所有估價單

### 3. 主控流程

- 檔案：`main.cpp`
- 內容：
  - 程式啟動
  - 主選單迴圈
  - 顯示功能並選擇
  - 依照功能做反應
    - 1：新增估價單
    - 2：關鍵字查詢 ( 客戶姓名 / 估價單日期 )
    - 3：刪除特定估價單 ( 列出後選擇編號 )
    - 4：顯示全部估價單
    - 5：離開程式

main.exe

## 四、功能操作說明

程式主選單如下：

1. [新增估價單](#1新增估價單)
2. [查詢估價單](#2查詢估價單)
3. [刪除估價單](#3刪除估價單)
4. [顯示所有估價單](#4顯示所有估價單)
5. [離開](#5離開系統)

### 1.新增估價單

#### 輸入欄位（依畫面順序）

1. 客戶姓名（不可空白）
2. 客戶稱謂/職稱（不可空白 例如：先生、小姐、經理、廠長）
3. 估價單日期（建議格式：`YYYY-MM-DD`）
4. 車輛年份（整數 限制在合理範圍）
5. 車輛品牌（不可空白）
6. 車輛型號（不可空白）
7. 服務內容（不可空白）
8. 零件費（`>= 0`）
9. 工資（`>= 0`）
10. 稅率（`>= 0` 輸入 5 代表 5%）

#### 系統處理流程

1. 逐欄讀取使用者輸入
2. 若輸入格式錯誤或超出範圍 立即提示重輸
3. 建立 `ServiceQuote` 物件
4. 指派系統流水號 `quoteId`
5. 儲存到 `QuoteManager` 內部清單
6. 顯示新增成功訊息與編號

#### 計算結果（自動）

- 小計 `subtotal = partsCharge + laborCharge`
- 稅金 `taxAmount = subtotal * taxRate / 100`
- 總估價 `totalAmount = subtotal + taxAmount`

### 2.查詢估價單

#### 使用方式

1. 從主選單選擇「2. 查詢估價單」
2. 輸入關鍵字（可輸入姓名片段或日期片段）
3. 系統巡覽所有估價單並比對符合項目

#### 比對規則

- 比對欄位：客戶姓名、估價單日期
- 字串包含即可成立 不需要完全相同
- 英文字母採不分大小寫比對（例如 `Tom` 與 `tom` 視為相同）

#### 輸出結果

- 若有結果：逐筆顯示完整摘要（編號、客戶資訊、車輛資訊、金額資訊）
- 若無結果：顯示「查無資料」類型提示

### 3.刪除估價單

#### 使用方式

1. 從主選單選擇「3. 刪除估價單」
2. 系統先列出目前所有估價單 方便確認目標編號
3. 輸入要刪除的 `quoteId`

#### 刪除規則

- 若 `quoteId` 存在：從容器中移除該筆資料
- 若 `quoteId` 不存在：不做刪除 回報失敗訊息

### 4.顯示所有估價單

#### 使用方式

1. 從主選單選擇「4. 顯示所有估價單」
2. 系統依目前儲存順序逐筆輸出

#### 顯示內容

- 估價單編號
- 客戶姓名
- 稱謂/職稱
- 日期
- 車輛年份
- 服務內容
- 零件費、工資、小計、稅率、稅金、總估價

#### 空清單情況

- 若目前尚無任何估價單 系統會提示使用者先新增資料

### 5.離開系統

1. 從主選單選擇「5. 離開」
2. 結束主迴圈並停止程式
3. 顯示結束訊息

## 五、特殊功能詳述

### 1. 輸入防呆與型別驗證

- 所有輸入都透過包裝函式讀取 降低 `cin` 讀取失敗造成流程中斷的風險
- 整數欄位（選單、年份）有格式與範圍檢查
- 小數欄位（零件費、工資、稅率）禁止負數
- 文字欄位預設禁止空白輸入
- 輸入錯誤時不會結束程式 而是提示後重輸

### 2. 關鍵字查詢彈性

- 支援「部分字串」查詢 提升實際使用便利性
- 英文查詢採大小寫不敏感
- 可用日期片段搜尋 例如輸入 `2026-04` 找出同月份資料

### 3. 金額自動計算與一致性

- 所有金額由 `ServiceQuote` 內部方法計算 避免不同頁面自行重算造成不一致
- 顯示摘要時統一輸出小計、稅金、總估價
- 計算公式集中管理 後續若要調整稅制只需修改核心邏輯

### 4. 估價單編號管理

- 新增資料時由 `QuoteManager` 自動給號
- 編號具唯一性 便於查詢與刪除
- 刪除後不會改動既有資料的編號

### 5. 模組化分層設計

- `QuoteSystemCore.*` 負責資料結構與商業邏輯
- `QuoteSystemUI.*` 負責互動流程與畫面輸出
- `main.cpp` 負責流程控制與功能分派
- 分層後易於維護、除錯與擴充（例如加入匯出檔案功能）
